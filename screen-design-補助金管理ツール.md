## 1. 設計前提

### プロダクトの位置づけ

農業補助金の **交付決定後〜実績報告完了** フェーズに特化した管理ツール。
申請前の情報収集・申請書作成（Granty の領域）は対象外。

### ユーザー

- **対象**: 農場主（経営者 = 現場作業者を兼ねる）
- **チーム規模**: 2〜5人（家族経営・小規模農業法人）
- **同時管理件数**: 4〜10件の補助金を並走

### 端末・操作環境

- **主端末**: スマホ中心（現場・事務所どちらでも使う）
- **ネットワーク**: 比較的安定している場所での使用を想定（事務作業時）

### 最大の痛み

> 複数補助金が同時進行する中で、**何を提出すればいいか・どの書類をもう出したか・まだ揃っていないのはどれか**が管理できていない。

### 成功の定義

> どの補助金が今どの状態で、何が足りないかが一目でわかる

### 差別化ポイント

農業補助金の「採択後管理」に特化した唯一のツール。
KSAS・アグリノートは日々の農作業記録が主目的で補助金管理機能がない。
Granty は申請前〜申請に強いが採択後が弱い。

## 2. 補助金のライフサイクルとカンバン列

このツールが扱う範囲:

```
（申請前〜申請は対象外）
        ↓
  ┌──────────────┐   ┌──────────────┐   ┌──────────┐
  │   実施中      │ → │   報告済み    │ → │  完了    │
  │（書類収集中） │   │（審査待ち）   │   │（受領済）│
  └──────────────┘   └──────────────┘   └──────────┘
```

| カンバン列 | 意味 | 主な作業 |
|-----------|------|---------|
| **実施中** | 交付決定済み・事業実施中 | 書類収集、チェックリスト消し込み |
| **報告済み** | 実績報告書を行政に提出済み | 審査結果の待機 |
| **完了** | 補助金受領、全手続き終了 | 参照・記録のみ |

### 「要対応」フラグ

列の分類とは別に、以下の場合はカードに ⚠️ バッジを表示する:
- 報告期限まで **30日以内** かつ書類が未完了
- 報告期限を **超過** している

## 3. 画面一覧とナビゲーション構造

### 画面一覧

| ID | 画面名 | MVP | 概要 |
|----|--------|-----|------|
| S-01 | ログイン | ✅ | チームコード + 名前で参加 |
| S-02 | 補助金一覧（カンバン） | ✅ | 全補助金をステータス別に一覧 ← メイン |
| S-03 | 補助金詳細 | ✅ | 1件の補助金の全情報・書類チェックリスト |
| S-04 | 書類登録・編集 | ✅ | 必要書類の追加・ステータス更新 |
| S-05 | 補助金追加 | ✅ | テンプレートから選択またはフリー入力 |
| S-06 | 期限一覧 | ✅ | 全補助金の期限を時系列で確認 |
| S-07 | 設定 | ✅ | チーム名・メンバー管理・ログアウト |

### ナビゲーション構造

スマホ: **ボトムナビゲーション 3タブ**

```
ログイン（S-01）
  └─ ホーム（ボトムナビ 3タブ）
       ├─ タブ1: 補助金一覧（S-02）
       │    ├─ 補助金詳細（S-03）
       │    │    └─ 書類登録・編集（S-04）
       │    └─ 補助金追加（S-05）
       ├─ タブ2: 期限一覧（S-06）
       └─ タブ3: 設定（S-07）
```

## 4. 各画面の仕様

---

### S-01 ログイン

**目的**: チームに参加して使い始める。認証を最小限にしてハードルを下げる。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│                         │
│   [ロゴ / アプリ名]      │
│   農業補助金かんり       │
│                         │
│   チームコードを入力     │
│   ┌─────────────────┐   │
│   │ XXXX-XXXX       │   │
│   └─────────────────┘   │
│                         │
│   あなたの名前          │
│   ┌─────────────────┐   │
│   │ 例: 田中 太郎    │   │
│   └─────────────────┘   │
│                         │
│   ┌─────────────────┐   │
│   │   参加する       │   │
│   └─────────────────┘   │
│                         │
│   ─── または ───        │
│                         │
│   新しいチームを作る →  │
│                         │
└─────────────────────────┘
```

**表示要素**
- アプリ名・ロゴ
- チームコード入力フィールド（8文字英数字）
- 名前入力フィールド
- 「参加する」ボタン（両方入力済みで活性化）
- 「新しいチームを作る」リンク

**インタラクション**
- 「参加する」タップ → チームコード検証 → S-02（補助金一覧）へ遷移
- 「新しいチームを作る」タップ → チーム名入力 → チームコード発行 → S-02 へ遷移
- 再訪時はチームコード・名前を復元してスキップ（ローカル保存）

**エッジケース**
- チームコード不正: 「チームが見つかりません」インラインエラー
- 名前未入力: 「参加する」ボタンを非活性

**カード表示要素**
- 補助金名
- 交付元（農水省 / 県 / 市町村）
- 報告期限（期限30日以内は赤表示）
- 書類進捗バー（完了件数 / 総件数）
- ⚠️ バッジ（期限30日以内 かつ 書類未完了）

**表示ロジック**
- 上部に「要対応」セクションを固定表示（⚠️バッジ案件のみ）
- カンバン列はスワイプで切り替え（実施中 / 報告済み / 完了）
- 実施中列: 報告期限の昇順（期限が近い順）でソート

**インタラクション**
- カードタップ → S-03（補助金詳細）へ遷移
- 右上「＋」タップ → S-05（補助金追加）へ遷移
- カードを左スワイプ → ステータスを次の列に進める（確認ダイアログあり）

**表示要素**
- ヘッダー情報（交付元・決定額・事業期間・報告期限）
- ステータス切り替えバー（実施中 / 報告済み / 完了）
- 書類チェックリスト（件数・進捗バー）
- 各書類アイテム（名称・ステータス・個別期限）
- フリーテキストメモ欄

**書類アイテムのステータス**

| ステータス | 表示 | 意味 |
|-----------|------|------|
| 未収集 | ☐ グレー | まだ書類を集めていない |
| 収集済み | ☐ ブルー | 手元に揃った、提出前 |
| 提出済み | ☑ グリーン | 行政に提出完了 |

**インタラクション**
- 書類アイテムタップ → S-04（書類登録・編集）へ遷移
- 「+追加」タップ → S-04（新規書類登録）へ遷移
- ステータスバーのタブタップ → ステータス変更（確認ダイアログあり）
- 「編集」タップ → 補助金基本情報の編集モード

**エッジケース**
- 書類が全件提出済みになったとき: 「実績報告書を提出しましたか？」確認バナーを表示
- ステータスを「完了」に変更: 「補助金を受領しましたか？」確認ダイアログ

### S-04 書類登録・編集

**目的**: 必要書類を追加し、収集状況を更新する。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│ ← 書類を登録            │
├─────────────────────────┤
│ 書類名 *                │
│ ┌─────────────────────┐ │
│ │ 例: 領収書（農機具） │ │
│ └─────────────────────┘ │
│                         │
│ ステータス *            │
│ ○ 未収集               │
│ ○ 収集済み             │
│ ○ 提出済み             │
│                         │
│ 個別期限（任意）        │
│ ┌─────────────────────┐ │
│ │ 2026/10/20          │ │
│ └─────────────────────┘ │
│                         │
│ ファイルリンク（任意）  │
│ ┌─────────────────────┐ │
│ │ Google Drive URL 等 │ │
│ └─────────────────────┘ │
│                         │
│ メモ（任意）            │
│ ┌─────────────────────┐ │
│ │ 原本が必要。写真不可 │ │
│ └─────────────────────┘ │
│                         │
│ ┌─────────────────────┐ │
│ │        保存         │ │
│ └─────────────────────┘ │
│                         │
│     [この書類を削除]    │
└─────────────────────────┘
```

**表示要素**
- 書類名（必須）
- ステータス選択（未収集 / 収集済み / 提出済み）
- 個別期限（任意：補助金全体の期限と別に設定できる）
- ファイルリンク（任意：Google Drive・Dropbox等のURL）
- メモ（任意）
- 削除ボタン（編集時のみ）

**エッジケース**
- 書類名が空白: 「保存」ボタンを非活性
- ファイルリンクが無効なURL: 「有効なURLを入力してください」エラー

## 5. データ構造（概略）

```
Team
  ├─ id, name, code
  └─ members[]

  ├─ id, team_id
  ├─ name（補助金名）

  └─ documents[]

  ├─ id, subsidy_id
  ├─ name（書類名）
  ├─ status（未収集 / 収集済み / 提出済み）
  ├─ deadline（個別期限、任意）
  ├─ file_url（外部リンク、任意）
  └─ memo
```
