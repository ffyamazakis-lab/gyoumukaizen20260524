## 1. 設計前提

### ユーザー
- **対象**: 個人農家〜農業法人（両方）
- **チーム規模**: 2〜5人（家族経営・小規模農業法人）
- **農業種別**: 水田・穀作を起点に設計

### 端末・操作環境
- **主端末**: スマホ中心（現場での入力を想定）
- **ネットワーク**: 農場では電波が弱い場合あり → オフライン時の考慮が必要

### 成功の定義
> 誰が何をやっているか一目でわかる（見える化）

### 差別化ポイント
- シンプルさ優先。アグリノート・クボタKSASHA より画面が少なく迷わない
- 既存ツールとのAPI連携なし（スタンドアロン）

### MVP スコープ（初期リリース）
| ペイン | 内容 | 対応 |
|--------|------|------|
| 1 | 今日のタスク一覧 | MVP |
| 2 | メンバーのスケジュール | MVP |
| 3 | 農場マップ | 将来対応 |
| 4 | 日報入力 | MVP |

### 将来対応（スコープ外）
- 農場マップ（ペイン3）
- 農薬・施肥の記録義務対応
- 補助金・行政書類との連携
- JAシステムとの連携

## 2. 画面一覧とナビゲーション構造

### 画面一覧

| ID | 画面名 | MVP | 概要 |
|----|--------|-----|------|
| S-01 | ログイン | ✅ | チームコード + 名前で参加（認証シンプル化） |
| S-02 | 今日のタスク一覧 | ✅ | 自分・チームの当日作業を確認、完了チェック |
| S-03 | メンバーのスケジュール | ✅ | 誰がどの農場で何をしているか一覧 |
| S-04 | 日報入力 | ✅ | 作業内容・時間・メモをサッと記録 |
| S-05 | 日報確認 | ✅ | 入力済み日報の参照・編集 |
| S-06 | タスク追加 | ✅ | 日付・農場・担当・作業内容を登録 |
| S-07 | 設定 | ✅ | チーム名・メンバー管理・ログアウト |

### ナビゲーション構造

スマホでの「4ペイン」= **ボトムナビゲーション 4タブ** として実現する。

```
ログイン（S-01）
  └─ ホーム（ボトムナビ 4タブ）
       ├─ タブ1: 今日のタスク（S-02）
       │    └─ タスク追加（S-06）
       ├─ タブ2: メンバースケジュール（S-03）
       ├─ タブ3: 日報（S-04）
       │    └─ 日報確認（S-05）
       └─ タブ4: 設定（S-07）
```

## 3. 各画面の仕様

---

### S-01 ログイン

**目的**: チームに参加して使い始める。認証を最小限にしてハードルを下げる。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│                         │
│   [ロゴ / アプリ名]      │
│                         │
│   チームコードを入力     │
│   ┌─────────────────┐   │
│   │ XXXX-XXXX       │   │
│   └─────────────────┘   │
│                         │
│   あなたの名前          │
│   ┌─────────────────┐   │
│   │ 例: 田中 太郎    │   │
│   └─────────────────┘   │
│                         │
│   ┌─────────────────┐   │
│   │   参加する       │   │
│   └─────────────────┘   │
│                         │
│   ─── または ───        │
│                         │
│   新しいチームを作る →  │
│                         │
└─────────────────────────┘
```

**表示要素**
- アプリ名・ロゴ
- チームコード入力フィールド（8文字英数字）
- 名前入力フィールド
- 「参加する」ボタン（チームコード + 名前が入力済みで活性化）
- 「新しいチームを作る」リンク（チーム作成フローへ）

**インタラクション**
- 「参加する」タップ → チームコード検証 → S-02（今日のタスク一覧）へ遷移
- 「新しいチームを作る」タップ → チーム名入力 → チームコード発行 → S-02 へ遷移
- 再訪時はチームコード・名前を復元してスキップ（ローカル保存）

**エッジケース**
- チームコード不正: 「チームが見つかりません」インラインエラー
- 名前未入力: 「参加する」ボタンを非活性
- オフライン時: 保存済み情報がある場合はオフラインモードで継続

### S-02 今日のタスク一覧

**目的**: その日にやるべき作業を確認し、完了したらチェックする。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│ ◀  今日のタスク  5/13  │
│                    [＋] │
├─────────────────────────┤
│ [ 自分 ]  [ チーム全員 ] │
├─────────────────────────┤
│ ☐  田んぼA 水管理       │
│    田中 太郎 ・ 午前     │
├─────────────────────────┤
│ ☑  田んぼB 草刈り       │
│    鈴木 花子 ・ 完了     │
├─────────────────────────┤
│ ☐  農機具 点検          │
│    田中 太郎 ・ 終日     │
├─────────────────────────┤
│                         │
│   （タスクなし時）       │
│   今日の予定はありません │
│   ＋ タスクを追加する   │
│                         │
├─────────────────────────┤
│ [タスク] [メンバー] [日報] [設定] │
└─────────────────────────┘
```

**表示要素**
- ヘッダー: 今日の日付、タスク追加ボタン（＋）
- セグメント切替: 「自分」「チーム全員」
- タスクカード（1件ごと）:
  - 完了チェックボックス
  - 作業内容
  - 担当者名・時間帯（午前 / 午後 / 終日）
  - 完了済みはグレーアウト + 打ち消し線
- ボトムナビゲーション（4タブ）

**インタラクション**
- チェックボックスタップ → 完了状態をトグル、他メンバーの画面にリアルタイム反映
- タスクカードタップ → 詳細表示（インラインまたは S-06 編集画面）
- 「＋」タップ → S-06（タスク追加）へ遷移
- セグメント切替 → 表示対象を絞り込み

**エッジケース**
- タスク0件: 「今日の予定はありません＋タスクを追加する」を表示
- オフライン: キャッシュ済みデータを表示、同期保留中バッジを表示
- 長いタスク名: 1行に収まらない場合は2行まで表示し折りたたみ

---

### S-03 メンバーのスケジュール

**目的**: チーム全員が今日どの農場で何をしているか一目で把握する。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│    メンバースケジュール  │
│                  5/13   │
├─────────────────────────┤
│ 田中 太郎               │
│   田んぼA 水管理  午前  │
│   農機具 点検     終日  │
├─────────────────────────┤
│ 鈴木 花子               │
│   田んぼB 草刈り  ✅完了 │
├─────────────────────────┤
│ 佐藤 次郎               │
│   （本日の予定なし）     │
├─────────────────────────┤
│ [タスク] [メンバー] [日報] [設定] │
└─────────────────────────┘
```

**表示要素**
- ヘッダー: 画面名・日付
- メンバーカード（1人ごと）:
  - メンバー名
  - その日のタスク一覧（農場・作業内容・時間帯・完了状態）
  - タスクなし時は「本日の予定なし」を薄く表示
- ボトムナビゲーション

**インタラクション**
- メンバーカードタップ → そのメンバーのタスク詳細を展開
- タスク行タップ → S-02 のそのタスクへジャンプ（スクロール）
- 日付ナビ: ヘッダーの日付を左右スワイプ or タップで前後の日に移動

**エッジケース**
- メンバー全員がタスクなし: 「今日の予定が登録されていません」を表示
- メンバーが6人以上: スクロールで対応（1画面に収まらない場合）
- オフライン: キャッシュ済みデータを表示

---

### S-04 日報入力

**目的**: 作業が終わったら、その場でサッと内容・時間・メモを記録する。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│  ◀  日報入力   5/13     │
├─────────────────────────┤
│ 作業を選ぶ              │
│ ┌─────────────────────┐ │
│ │ 田んぼA 水管理    ▼ │ │
│ └─────────────────────┘ │
│ （今日のタスクから選択） │
├─────────────────────────┤
│ 実施時間                │
│  開始  [08:00]  終了  [11:30] │
├─────────────────────────┤
│ 作業内容・メモ          │
│ ┌─────────────────────┐ │
│ │ 例: 水位を確認。問題  │ │
│ │ なし。               │ │
│ └─────────────────────┘ │
├─────────────────────────┤
│ 天気                    │
│  ☀晴 ⛅曇 🌧雨 [選択]  │
├─────────────────────────┤
│ ┌─────────────────────┐ │
│ │     日報を保存する   │ │
│ └─────────────────────┘ │
├─────────────────────────┤
│  過去の日報を見る →     │
├─────────────────────────┤
│ [タスク] [メンバー] [日報] [設定] │
└─────────────────────────┘
```

**表示要素**
- ヘッダー: 日付、戻るボタン
- 作業選択: 今日のタスク一覧からプルダウン選択（手入力も可）
- 時間入力: 開始・終了時刻（時刻ピッカー）
- メモ: 自由記述テキストエリア（任意）
- 天気: アイコン選択（晴・曇・雨）
- 「日報を保存する」ボタン
- 「過去の日報を見る」リンク → S-05 へ

**インタラクション**
- 作業プルダウン: S-02 の今日のタスクが選択肢として並ぶ
- 「日報を保存する」タップ → 保存 → S-02 の対象タスクを完了状態に更新 → 完了トースト表示
- 「過去の日報を見る」タップ → S-05（日報確認）へ遷移

**エッジケース**
- 今日のタスクがない場合: 作業内容を手入力で対応
- 同じタスクに2件目の日報: 「すでに記録があります。追記しますか？」確認
- オフライン: ローカル保存 → 同期保留中バッジ表示 → オンライン復帰時に自動送信

### S-05 日報確認

**目的**: 過去の日報を日付・メンバー別に参照・編集する。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│  ◀  日報一覧            │
│  [  自分  ] [チーム全員] │
├─────────────────────────┤
│ 2026/05/13              │
│ ─────────────────────── │
│  田んぼA 水管理          │
│  08:00〜11:30  田中太郎 │
│  晴 / 水位確認、問題なし │
│                   [編集] │
├─────────────────────────┤
│ 2026/05/12              │
│ ─────────────────────── │
│  田んぼB 草刈り          │
│  09:00〜15:00  鈴木花子 │
│  曇 / 全体3/4 完了      │
│                   [編集] │
├─────────────────────────┤
│ [タスク] [メンバー] [日報] [設定] │
└─────────────────────────┘
```

**表示要素**
- ヘッダー: 戻るボタン・タイトル
- セグメント: 「自分」「チーム全員」
- 日付グループ（降順）
- 日報カード: 作業名・時間・担当者・天気・メモ抜粋・編集ボタン

**インタラクション**
- 「編集」タップ → S-04（日報入力）を編集モードで開く
- スクロールで過去日付を遡る
- セグメント切替でフィルタ

**エッジケース**
- 日報0件: 「まだ日報がありません」を表示
- 長いメモ: 2行で切り捨て、タップで全文表示

### S-06 タスク追加

**目的**: 特定の日付・農場・担当者に作業を登録する。

**ワイヤーフレーム**

```
┌─────────────────────────┐
│  ◀  タスクを追加        │
├─────────────────────────┤
│ 日付                    │
│  ┌─────────────────┐    │
│  │ 2026/05/13    ▼ │    │
│  └─────────────────┘    │
├─────────────────────────┤
│ 農場・場所              │
│  ┌─────────────────┐    │
│  │ 田んぼA       ▼ │    │
│  └─────────────────┘    │
├─────────────────────────┤
│ 担当者                  │
│  ┌─────────────────┐    │
│  │ 田中 太郎     ▼ │    │
│  └─────────────────┘    │
├─────────────────────────┤
│ 作業内容                │
│  ┌─────────────────┐    │
│  │ 例: 水管理      │    │
│  └─────────────────┘    │
├─────────────────────────┤
│ 時間帯                  │
│  [午前]  [午後]  [終日]  │
├─────────────────────────┤
│ メモ（任意）            │
│  ┌─────────────────┐    │
│  │                 │    │
│  └─────────────────┘    │
├─────────────────────────┤
│  ┌─────────────────┐    │
│  │   タスクを登録   │    │
│  └─────────────────┘    │
└─────────────────────────┘
```

**表示要素**
- ヘッダー: 戻るボタン・タイトル
- 日付選択（カレンダーピッカー、デフォルト: 今日）
- 農場・場所（登録済みリストからプルダウン、新規入力も可）
- 担当者（チームメンバーからプルダウン）
- 作業内容（テキスト入力）
- 時間帯（午前 / 午後 / 終日 の3択）
- メモ（任意・テキストエリア）
- 「タスクを登録」ボタン

**インタラクション**
- 「タスクを登録」タップ → 保存 → S-02（今日のタスク一覧）に戻る
- 農場が未登録の場合: 「新しい農場を追加」でインライン登録

**エッジケース**
- 作業内容が未入力: ボタンを非活性
- 同一担当・同一日・同一農場に重複登録: 警告なしで許容（複数作業あり得る）

**目的**: チーム・メンバー管理とアカウント操作。

**ワイヤーフレーム**

```
┌─────────────────────────┐

├─────────────────────────┤

│  佐藤 次郎              │
│  ＋ メンバーを招待する  │
├─────────────────────────┤
│ 農場・場所の管理        │
│  田んぼA / 田んぼB / …  │
│  ＋ 農場を追加する      │
├─────────────────────────┤
│  ─────────────────────  │
│  ログアウト             │
├─────────────────────────┤
│ [タスク] [メンバー] [日報] [設定] │
└─────────────────────────┘
```

**表示要素**
- チームセクション: チーム名・チームコード表示、コードコピーボタン
- メンバーセクション: メンバー一覧、招待リンク（チームコードをシェア）
- 農場管理: 登録済み農場一覧、追加ボタン

**インタラクション**
- 「コードをコピー」タップ → クリップボードにコピー + トースト表示

- 「農場を追加する」タップ → 農場名入力ダイアログ
- 「ログアウト」タップ → 確認ダイアログ → S-01（ログイン）へ遷移

**エッジケース**
- メンバーが自分1人: 「チームコードを共有して仲間を招待しましょう」を促す
- 農場が未登録: S-06（タスク追加）時に誘導

## 5. 共通ルール

### ローディング
- データ取得中: スケルトンUIを表示（空白より体感速度が上がる）
- 操作後の保存中: ボタンをローディングスピナーに差し替え

### エラー表示
- ネットワークエラー: 画面上部にバナー「オフラインです。データは同期保留中です。」
- バリデーションエラー: 入力フィールド直下にインラインテキスト

### データがない場合（空状態）
- イラストなし・テキストのみでシンプルに
- 必ずアクションへの誘導（「＋ タスクを追加する」等）を添える

### フォント・色
- 使用ツール・フレームワーク確定後に別途定義
- 原則: 黒・白・グレー + アクセント1色（緑系 = 農業らしさ）
